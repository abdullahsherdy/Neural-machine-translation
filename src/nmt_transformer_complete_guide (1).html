Why Pre-Norm instead of Post-Norm?

The original "Attention Is All You Need" paper used Post-Norm (LayerNorm after the sublayer). Pre-Norm (LayerNorm before) is now standard because it prevents gradient vanishing/exploding through deep stacks — you can train 6 layers stably without warmup tricks.

Weight tying explained

The final linear layer that maps 512-dim → 16,000 Arabic vocab logits shares its weight matrix with the Arabic embedding layer. This means the same matrix is used both to look up "what does token 4821 mean as input?" and "how likely is token 4821 as the next output?". This saves ~8 M parameters at D=512 and consistently improves BLEU by +0.3–0.5 points.

self.fc_out = nn.Linear(d_model, tgt_vocab, bias=False)
self.fc_out.weight = self.decoder.embedding.weight  # weight tying

BLEU Score — explained from scratch

📐
One sentence summary: BLEU measures how many n-grams (word sequences) in your predicted translation also appear in the reference translation.

Step-by-step calculation

Suppose the reference is "the meeting was postponed" and your model outputs "the meeting got postponed".

n-gram orderReference n-gramsHypothesis n-gramsMatchesPrecision
1-gram (words)the, meeting, was, postponedthe, meeting, got, postponed3/40.75
2-gram (pairs)the meeting, meeting was, was postponedthe meeting, meeting got, got postponed1/30.33
3-gramthe meeting was, meeting was postponedthe meeting got, meeting got postponed0/20.00
4-gramthe meeting was postponedthe meeting got postponed0/10.00
BLEU-4 formula
BLEU = BP × exp( (1/4) × Σ log(pₙ) ) where n = 1..4

BP (Brevity Penalty) = min(1.0, exp(1 - reference_length / hypothesis_length))
pₙ = clipped_matches_of_order_n / hypothesis_ngrams_of_order_n

Intuition: geometric mean of 1-gram, 2-gram, 3-gram, 4-gram precision.
BP penalises translations that are too short (easy way to get high precision).

⚠️ BLEU limitations for Arabic specifically
Arabic is morphologically very rich — a single root like كتب can produce كتب، كتاب، مكتبة، كاتب، مكتوب. BLEU counts exact n-gram matches, so if your model generates a correct but differently-inflected form, it gets zero credit. Arabic BLEU scores are typically 3–5 points lower than English for the same quality of translation. Token accuracy (which you can see per-epoch) is a more reliable training signal.

Greedy vs Beam search BLEU

Your notebook computes both. Greedy decoding always picks the single highest-probability token at each step. Beam search (width=5) maintains 5 partial hypotheses and picks the globally best sequence — it's slower but consistently gives +2–4 BLEU points over greedy.

# Greedy: fast, slightly lower BLEU
g_ids, attn_map = model.translate_greedy(src_tensor)

# Beam-5: slower, best quality, length-penalty α=0.6
b_ids = model.translate_beam(src_tensor, beam_width=5, alpha=0.6)

Perplexity (PPL) — explained

🎲
One sentence summary: Perplexity is "how many equally likely choices does the model feel it has at each step?" Lower = more certain = better.
Perplexity from cross-entropy loss
PPL = exp(CE_loss)

If CE loss = 4.0 → PPL = e⁴ = 54.6 (model is confused between ~55 equally likely tokens)
If CE loss = 3.0 → PPL = e³ = 20.1 (model has narrowed to ~20 candidates)
If CE loss = 2.3 → PPL = e²·³ = 10.0 (very confident — 10 candidates)
If CE loss = 1.0 → PPL = e¹ = 2.7 (near-perfect — deciding between ~3 tokens)

Target for good translation: PPL < 20 (CE loss < ~3.0)

Loss functions in this notebook

Cross-Entropy Loss

The base loss. For each token position, it computes: "the negative log-probability assigned to the correct Arabic token".

CE_loss(position t) = -log( softmax(logits)[correct_token_id] )

If model assigns 90% prob to correct token → loss = -log(0.90) = 0.105 ✅ good
If model assigns 10% prob to correct token → loss = -log(0.10) = 2.302 ❌ bad
If model assigns 0.1% prob (early training) → loss = -log(0.001) = 6.9

Final loss = mean over all non-PAD token positions in the batch

Label Smoothing (ε = 0.1)

Instead of the model targeting a hard 100% probability on the correct token, label smoothing distributes 10% of the probability mass evenly across all other tokens. This prevents overconfidence and improves generalisation (typically +0.3–0.5 BLEU).

Without smoothing: target = [0, 0, 0, 1.0, 0, ...] for correct token at index 3
With smoothing ε=0.1: target = [ε/(V-1), ..., 1-ε, ..., ε/(V-1)]
= [0.0000063, ..., 0.9, ..., 0.0000063]

Effective: model is penalised less for being wrong on rare tokens,
but also gets less credit for being 100% confident on common ones.

Noam Learning Rate Schedule

Named after Noam Shazeer (Google Brain). Increases LR linearly for warmup_steps steps, then decays proportional to the inverse square root of step number.

lr(step) = d_model^(-0.5) × min(step^(-0.5), step × warmup^(-1.5))

With D=256, warmup=4000:
Peak LR at step 4000 = 256^(-0.5) × 4000^(-0.5) = 0.0625 × 0.0158 ≈ 0.00099

Why warmup? At step 1, gradients are random. A large LR would destroy the
embedding initialisation. Warmup starts tiny, letting the model find a stable
gradient direction before the LR climbs.

What the curves show

  • Steep early descent (epochs 1–10): val loss drops from 7.81 → 4.49 as Noam LR rises through warmup (3.6e-5 → 3.6e-4) and the model learns the BPE vocabulary distribution.
  • Slow refinement (epochs 11–18): loss keeps falling smoothly from 4.40 → 4.05; accuracy rises 39 → 45 %. Cross-attention is sharpening into translation alignments.
  • Plateau (epochs 19–26): val loss oscillates around 4.05–4.10 and accuracy hovers at ~45–46 %. Train loss keeps falling (3.24 → 2.82) — small but real over-fitting opens up.
  • Early stopping kicks in 5 epochs after the best val loss (epoch 21), correctly saving the checkpoint just before the divergence.
